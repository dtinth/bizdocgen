# คำถามที่พบบ่อย (FAQ)

รวบรวมคำถามและคำตอบที่พบบ่อย

## การติดตั้งและเริ่มต้นใช้งาน

### ถาม: ต้องเสียค่าใช้จ่ายไหม?

**ตอบ:** 
- **bizdocgen**: ฟรี (MIT License)
- **Grist Free Tier**: ฟรี จำกัด 5,000 แถวต่อเอกสาร
- **Grist Paid**: มีแผนการตั้งแต่ $8/เดือน

### ถาม: ใช้กับ Grist ที่ host เองได้ไหม?

**ตอบ:** ได้ bizdocgen รองรับทั้ง Hosted Grist และ Self-Hosted Grist

### ถาม: ต้องมีความรู้ทางเทคนิคหรือไม่?

**ตอบ:** ไม่จำเป็น ถ้าใช้ Hosted Grist เพียงแค่คลิกใช้เทมเพลต

### ถาม: ข้อมูลจะปลอดภัยหรือไม่?

**ตอบ:** 
- ข้อมูลจัดเก็บใน Grist ของคุณ
- widget ไม่ส่งข้อมูลไปยัง server ภายนอก
- QR Code สร้างใน browser โดยตรง

## การใช้งาน

### ถาม: เปลี่ยนภาษาได้ไหม?

**ตอบ:** ปัจจุบันรองรับภาษาไทยและอังกฤษผสมกัน ตามมาตรฐานเอกสารไทย

### ถาม: เพิ่มประเภทเอกสารเองได้ไหม?

**ตอบ:** ต้องแก้ไขโค้ด (Credit Note, Delivery Order ฯลฯ) สามารถ fork และ customize ได้

### ถาม: รองรับหลายสกุลเงินหรือไม่?

**ตอบ:** ปัจจุบันรองรับ THB (บาท) เท่านั้น

### ถาม: พิมพ์เป็น PDF ได้ไหม?

**ตอบ:** ได้ เลือก "Save as PDF" ใน dialog พิมพ์ของ browser

### ถาม: ส่งออกเป็น Excel ได้ไหม?

**ตอบ:** Grist รองรับ export เป็น CSV ซึ่งเปิดใน Excel ได้

## การปรับแต่ง

### ถาม: เปลี่ยน logo ได้ไหม?

**ตอบ:** ใช้ CSS ซ่อน header เดิมและเพิ่ม logo ด้วย `::before` หรือใช้ background-image

```css
.document-header__provider-name::before {
  content: '';
  display: block;
  width: 100px;
  height: 50px;
  background: url('data:image/png;base64,...') no-repeat;
  margin-bottom: 1rem;
}
```

### ถาม: เปลี่ยนฟอนต์ได้ไหม?

**ตอบ:** ได้ ผ่าน Custom CSS:

```css
@import url('https://fonts.googleapis.com/css2?family=Prompt&display=swap');

.document {
  --font-family: 'Prompt', sans-serif;
}
```

### ถาม: ปรับขนาดกระดาษได้ไหม?

**ตอบ:** ได้ แต่ต้องระวังเรื่องการพิมพ์:

```css
.document {
  --document-width: 210mm;  /* A4 */
  --document-height: 297mm;
}
```

### ถาม: CSS ที่ตั้งค่าจะหายไหม?

**ตอบ:** ถ้าใช้ Hosted Grist CSS จะถูกบันทึกใน widget options ถ้าใช้ Self-hosted ขึ้นอยู่กับการตั้งค่า

## ภาษีและการคำนวณ

### ถาม: คำนวณภาษีถูกต้องหรือไม่?

**ตอบ:** ระบบใช้การคำนวณมาตรฐาน:
- VAT = ยอดรวม × 0.07
- WHT = ยอดรวม × อัตรา WHT (ติดลบ)

### ถาม: รองรับ WHT กี่ %?

**ตอบ:** รองรับทุกอัตรา โดยใส่ค่าเป็นลบ:
- 1% = -0.01
- 2% = -0.02
- 3% = -0.03
- 5% = -0.05

### ถาม: ทำไมภาษีไม่แสดง?

**ตอบ:** ตรวจสอบ:
- ค่า Tax ต้องเป็นตัวเลข (ไม่ใช่ข้อความ)
- ค่าต้องอยู่ระหว่าง -1 ถึง 1

### ถาม: คำนวณ VAT ก่อนหรือหลัง WHT?

**ตอบ:** ระบบคำนวณตามค่า Tax ที่ตั้งไว้เท่านั้น ถ้าต้องการ VAT + WHT ต้องคำนวณเองในรายการสินค้า

## PromptPay และ QR Code

### ถาม: PromptPay ใช้ได้กับทุกธนาคารไหม?

**ตอบ:** ใช่ QR Code ที่สร้างเป็นมาตรฐาน PromptPay ทุกธนาคารรองรับ

### ถาม: QR Code สแกนไม่ได้?

**ตอบ:** ตรวจสอบ:
- หมายเลข PromptPay ถูกต้อง (10 หรือ 13 หลัก)
- ไม่มีอักขระพิเศษ
- QR Code ชัดเจน (ไม่เบลอ)

### ถาม: จำนวนเงินใน QR Code ถูกต้องไหม?

**ตอบ:** ใช่ ระบบคำนวณจากยอดสุทธิของเอกสารอัตโนมัติ

### ถาม: ใช้พร้อมกันหลายบัญชีได้ไหม?

**ตอบ:** ได้ สร้างหลาย Payment Method ใน Grist แล้วเลือกใช้ตามเอกสาร

## การพัฒนา

### ถาม: จะ contribute ได้ไหม?

**ตอบ:** ได้! เปิด Pull Request ที่ GitHub Repository

### ถาม: ใช้ภาษาอะไรพัฒนา?

**ตอบ:** 
- Vue 3 + TypeScript (Frontend)
- Zod (Validation)
- Playwright (E2E Testing)

### ถาม: มี API สำหรับ integrate ไหม?

**ตอบ:** ไม่มี API แยก แต่สามารถใช้ข้อมูลจาก Grist API ได้โดยตรง

### ถาม: ติดต่อนักพัฒนาได้อย่างไร?

**ตอบ:** เปิด Issue ที่ [GitHub Issues](https://github.com/dtinth/bizdocgen/issues)

## ปัญหาที่พบบ่อย

### เอกสารไม่แสดง

**สาเหตุ:**
- Widget ไม่มีสิทธิ์เข้าถึงข้อมูล
- Schema validation ล้มเหลว

**แก้ไข:**
- ตั้งค่า Access เป็น "Full document access"
- ตรวจสอบข้อมูลตรงกับ schema

### การพิมพ์ไม่สมบูรณ์

**สาเหตุ:**
- การตั้งค่า browser
- CSS ไม่เหมาะสม

**แก้ไข:**
- ปิด Header/Footer ในการตั้งค่าพิมพ์
- ตรวจสอบ @media print CSS

### ฟอนต์ไทยเพี้ยน

**สาเหตุ:**
- Browser ไม่รองรับฟอนต์
- การตั้งค่า encoding

**แก้ไข:**
- ใช้ Chrome หรือ Firefox
- ตั้งค่า encoding เป็น UTF-8

### ข้อมูลไม่อัพเดท

**สาเหตุ:**
- Grist ไม่ส่งข้อมูลใหม่
- Widget ค้าง

**แก้ไข:**
- รีโหลดหน้า
- ตรวจสอบ browser console ดู errors
