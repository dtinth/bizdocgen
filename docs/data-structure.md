# โครงสร้างข้อมูล

bizdocgen รับข้อมูลจาก Grist ในรูปแบบ Record ที่มีโครงสร้างดังนี้

## โครงสร้างหลัก (GristRecord)

```
GristRecord
├── id                          — รหัสแถว (number)
└── Record
    ├── Document_Type           — ประเภทเอกสาร
    ├── Number                  — เลขที่เอกสาร
    ├── Date                    — วันที่ (ISO 8601)
    ├── Provider                — ข้อมูลผู้ออกเอกสาร
    ├── Client                  — ข้อมูลลูกค้า
    ├── Items[]                 — รายการสินค้า/บริการ
    ├── Tax                     — อัตราภาษี
    ├── Payment_Method          — วิธีการชำระเงิน (ถ้ามี)
    ├── Reference               — เลขอ้างอิง (ถ้ามี)
    ├── Credit_Term             — เครดิตเทอม (ถ้ามี)
    ├── Remarks                 — หมายเหตุ (ถ้ามี)
    └── Signed_Document_URL     — URL เอกสารที่ลงชื่อแล้ว (ถ้ามี)
```

## รายละเอียดแต่ละฟิลด์

### Document_Type — ประเภทเอกสาร

ค่าที่รองรับ:

| ค่า | ความหมาย |
|-----|----------|
| `Quotation` | ใบเสนอราคา |
| `Invoice` | ใบแจ้งหนี้ |
| `Receipt` | ใบเสร็จรับเงิน |

!!! note
    ข้อมูลจาก Grist อาจส่งมาเป็นค่าเดี่ยว (`"Invoice"`) หรืออาร์เรย์ (`["Invoice"]`) ระบบจะ Normalize ให้เป็นอาร์เรย์อัตโนมัติ

### Provider — ข้อมูลผู้ออกเอกสาร

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |
|-------|--------|--------|----------|
| `Name` | string | ✅ | ชื่อธุรกิจ/แบรนด์ |
| `Address` | string | ✅ | ที่อยู่ (รองรับขึ้นบรรทัดใหม่) |
| `Tax_ID` | string | ✅ | เลขประจำตัวผู้เสียภาษี (13 หลัก) |
| `Email` | string | — | อีเมลติดต่อ |
| `Personnel_Name` | string | — | ชื่อผู้ลงนาม (สำหรับช่องลายเซ็น) |

### Client — ข้อมูลลูกค้า

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |
|-------|--------|--------|----------|
| `Name` | string | ✅ | ชื่อลูกค้า/บริษัท |
| `Address` | string | ✅ | ที่อยู่ |
| `Tax_ID` | string | ✅ | เลขประจำตัวผู้เสียภาษี |

### Items — รายการสินค้า/บริการ

แต่ละรายการมีฟิลด์ดังนี้:

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |
|-------|--------|--------|----------|
| `id` | number | ✅ | รหัสรายการ |
| `Description` | string | ✅ | รายละเอียด (รองรับ Markdown) |
| `Quantity` | number | ✅ | จำนวน |
| `Unit_Price` | number | ✅ | ราคาต่อหน่วย |
| `Total` | number | ✅ | ราคารวม |
| `Manual_Sort` | number | — | ลำดับการจัดเรียง (น้อย→มาก) |

!!! tip "การจัดเรียงรายการ"
    รายการจะเรียงตามค่า `Manual_Sort` จากน้อยไปมาก หากไม่กำหนดค่า จะใช้ค่าเริ่มต้นเป็น 0

### Tax — อัตราภาษี

| ค่า | ความหมาย | ตัวอย่างผลลัพธ์ |
|-----|----------|----------------|
| `0.07` | VAT 7% | ยอดรวม + 7% |
| `0` | ไม่มีภาษี | แสดงเฉพาะยอดรวม |
| `-0.03` | หัก ณ ที่จ่าย 3% | ยอดรวม − 3% |
| `-0.05` | หัก ณ ที่จ่าย 5% | ยอดรวม − 5% |

### Payment_Method — วิธีการชำระเงิน

ฟิลด์นี้ไม่จำเป็นต้องมี (nullable):

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |
|-------|--------|--------|----------|
| `Name` | string | ✅ | ชื่อวิธีการชำระเงิน (ใช้ภายใน) |
| `Bank` | string | — | ชื่อธนาคาร |
| `Branch` | string | — | สาขา |
| `Account_Number` | string | — | เลขที่บัญชี |
| `Account_Holder` | string | — | ชื่อบัญชี |
| `PromptPay` | string | — | หมายเลขพร้อมเพย์ (เบอร์โทร 10 หลัก หรือเลขผู้เสียภาษี) |

!!! info "QR Code พร้อมเพย์"
    QR Code จะแสดงเฉพาะเมื่อกำหนดค่า `PromptPay` เท่านั้น

### Reference — เลขอ้างอิง

ฟิลด์นี้ไม่จำเป็นต้องมี:

| ฟิลด์ | ประเภท | จำเป็น | คำอธิบาย |
|-------|--------|--------|----------|
| `Number` | string | ✅ | เลขที่เอกสารอ้างอิง |

### Signed_Document_URL — URL เอกสารที่ลงชื่อ

เป็น URL ที่ถูกต้อง หรือค่าว่าง (`""`) เมื่อกำหนดค่า URL ที่ไม่ว่าง ระบบจะแสดงลิงก์ดาวน์โหลดแทนตัวอย่างเอกสาร

## การตรวจสอบข้อมูล (Validation)

bizdocgen ใช้ [Zod](https://zod.dev/) ในการตรวจสอบข้อมูลแบบ Runtime:

- **ฟิลด์ที่จำเป็น** — หากขาดหายไป จะแสดงข้อความข้อผิดพลาด
- **ประเภทข้อมูล** — ตรวจสอบว่าค่าเป็นประเภทที่ถูกต้อง (string, number ฯลฯ)
- **ค่าที่กำหนด** — Document_Type ต้องเป็นค่าที่กำหนดเท่านั้น
- **URL ที่ถูกต้อง** — Signed_Document_URL ต้องเป็น URL ที่ถูกต้องหรือค่าว่าง

เมื่อข้อมูลไม่ผ่านการตรวจสอบ ระบบจะแสดงข้อความข้อผิดพลาดพร้อมรายละเอียดว่าฟิลด์ใดมีปัญหา
