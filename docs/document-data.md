# โครงสร้างข้อมูลเอกสาร

bizdocgen รับข้อมูลหลักจากเรคอร์ดของ Grist หนึ่งแถว โดยคาดหวังรูปแบบข้อมูลตาม schema ที่กำหนดในแอป

## โครงสร้างระดับบนสุด

| ฟิลด์ | ชนิดข้อมูล | คำอธิบาย |
| --- | --- | --- |
| `id` | number | รหัสของเรคอร์ดใน Grist |
| `Record` | object | ข้อมูลเอกสารทั้งหมด |

## ฟิลด์ภายใต้ `Record`

| ฟิลด์ | จำเป็น | คำอธิบาย |
| --- | --- | --- |
| `Client` | ต้องมี | ข้อมูลลูกค้า |
| `Credit_Term` | ไม่บังคับ | เครดิตเทอม เช่น `30 วัน` |
| `Date` | ต้องมี | วันที่เอกสารในรูปแบบ ISO date string |
| `Document_Type` | ต้องมี | รายการที่มีค่าเดียว: `Quotation`, `Invoice`, หรือ `Receipt` |
| `Items` | ต้องมี | รายการสินค้า/บริการ |
| `Number` | ต้องมี | เลขที่เอกสาร |
| `Payment_Method` | ไม่บังคับ | วิธีชำระเงิน |
| `Provider` | ต้องมี | ข้อมูลผู้ออกเอกสาร |
| `Reference` | ไม่บังคับ | เอกสารอ้างอิง เช่น PO หรือเลขใบแจ้งหนี้ |
| `Remarks` | ไม่บังคับ | หมายเหตุท้ายเอกสาร รองรับ Markdown |
| `Tax` | ต้องมี | อัตราภาษี เช่น `0.07`, `-0.03`, `0` |
| `Signed_Document_URL` | ไม่บังคับ | URL ของเอกสารที่ลงชื่อแล้ว |

## รายละเอียดข้อมูลย่อย

### Client

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| `Name` | ชื่อลูกค้า |
| `Address` | ที่อยู่ลูกค้า |
| `Tax_ID` | เลขประจำตัวผู้เสียภาษี |

### Provider

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| `Name` | ชื่อผู้ให้บริการหรือแบรนด์ |
| `Address` | ที่อยู่ผู้ออกเอกสาร |
| `Tax_ID` | เลขประจำตัวผู้เสียภาษี |
| `Email` | อีเมลติดต่อ |
| `Personnel_Name` | ชื่อผู้ลงนาม |

### Payment_Method

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| `Name` | ชื่อเรียกวิธีชำระเงิน ใช้ภายในระบบ |
| `Account_Holder` | ชื่อเจ้าของบัญชี |
| `Account_Number` | เลขบัญชี |
| `Bank` | ธนาคาร |
| `Branch` | สาขา |
| `PromptPay` | เบอร์โทรศัพท์หรือเลข PromptPay สำหรับสร้าง QR |

### Items

แต่ละรายการใน `Items` มีฟิลด์ดังนี้

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| `id` | รหัสรายการ |
| `Description` | รายละเอียดสินค้า/บริการ รองรับ Markdown |
| `Manual_Sort` | ลำดับการแสดงผล |
| `Quantity` | จำนวน |
| `Unit_Price` | ราคาต่อหน่วย |
| `Total` | ราคารวมของรายการ |

## หลักการแสดงผลสำคัญ

- รายการสินค้าเรียงตาม `Manual_Sort`
- วันที่แสดงผลเป็นรูปแบบภาษาไทย
- จำนวนเงินแสดงผลเป็นสกุลเงินบาท และมีข้อความจำนวนเงินภาษาไทย
- `Remarks` รองรับหัวข้อ รายการย่อย และการจัดรูปแบบแบบ Markdown

## ตัวอย่างแนวคิดของข้อมูล

```json
{
  "id": 1,
  "Record": {
    "Document_Type": ["Invoice"],
    "Number": "INV-2025-0001",
    "Date": "2025-08-09T00:00:00.000Z",
    "Tax": 0.07
  }
}
```

!!! warning
    ถ้าข้อมูลไม่ผ่าน schema ของแอป วิดเจ็ตจะแสดงข้อความผิดพลาดแทนเอกสาร เช่น กรณีไม่มีฟิลด์สำคัญหรือชนิดข้อมูลไม่ตรง
