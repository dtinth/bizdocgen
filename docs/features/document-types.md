# ประเภทเอกสาร

bizdocgen รองรับการสร้างเอกสารทางธุรกิจ 3 ประเภทหลัก ตามมาตรฐานไทย

## ใบเสนอราคา (Quotation)

เอกสารสำหรับเสนอราคาสินค้าหรือบริการให้กับลูกค้า

### ส่วนประกอบ

- **เลขที่เอกสาร** - หมายเลขอ้างอิง (เช่น QT-2024-001)
- **วันที่** - วันที่ออกเอกสาร
- **ข้อมูลผู้ให้บริการ** - ชื่อ ที่อยู่ เลขประจำตัวผู้เสียภาษี
- **ข้อมูลลูกค้า** - ชื่อ ที่อยู่ เลขประจำตัวผู้เสียภาษี
- **รายการสินค้า/บริการ** - รายละเอียด จำนวน ราคาต่อหน่วย ราคารวม
- **สรุปยอด** - ราคารวม ภาษี ยอดสุทธิ
- **เงื่อนไขการชำระ** - เช่น 30 วัน
- **หมายเหตุ** - ข้อความเพิ่มเติม

### ตัวอย่างการใช้งาน

```markdown
เลขที่: QT-2024-001
วันที่: 15 มกราคม 2024
ลูกค้า: บริษัท ตัวอย่าง จำกัด

รายการ:
1. บริการออกแบบเว็บไซต์ - 1 งาน - 50,000 บาท
2. บริการพัฒนาระบบ - 1 งาน - 80,000 บาท

รวม: 130,000 บาท
VAT 7%: 9,100 บาท
รวมสุทธิ: 139,100 บาท

เงื่อนไข: ชำระภายใน 30 วัน
```

## ใบแจ้งหนี้ (Invoice)

เอกสารสำหรับแจ้งหนี้เรียกเก็บเงินจากลูกค้า

### ส่วนประกอบ

ครอบคลุมทุกส่วนของใบเสนอราคา พร้อมเพิ่ม:

- **อ้างอิงใบเสนอราคา** - เชื่อมโยงกับ QT (optional)
- **ข้อมูลการชำระเงิน** - บัญชีธนาคาร PromptPay
- **QR Code** - สำหรับสแกนชำระเงิน

### ตัวอย่างการใช้งาน

```markdown
เลขที่: INV-2024-001
วันที่: 1 กุมภาพันธ์ 2024
อ้างอิง: QT-2024-001

รายการเหมือนใบเสนอราคา

ชำระเงินได้ที่:
- PromptPay: 091-234-5678
- ธนาคารกสิกรไทย สาขาสุขุมวิท
  บัญชี: 123-4-56789-0
  ชื่อบัญชี: บริษัท ผู้ให้บริการ จำกัด
```

## ใบเสร็จรับเงิน (Receipt)

เอกสารยืนยันการรับชำระเงินจากลูกค้า

### ส่วนประกอบ

- **เลขที่เอกสาร** - หมายเลขอ้างอิง (เช่น RC-2024-001)
- **อ้างอิงใบแจ้งหนี้** - เชื่อมโยงกับ INV (optional)
- **ส่วนลงชื่อ** - ช่องลงชื่อผู้รับเงินและวันที่
- **ข้อมูลการชำระ** - วิธีการชำระที่ลูกค้าใช้

### ตัวอย่างการใช้งาน

```markdown
เลขที่: RC-2024-001
วันที่: 15 กุมภาพันธ์ 2024
อ้างอิบ: INV-2024-001

ได้รับเงินจาก: บริษัท ตัวอย่าง จำกัด
จำนวน: 139,100 บาท

ลงชื่อ _________________ ผู้รับเงิน
      (นาย โปร แก้ได้หมด)

วันที่ _____/_____/______
```

## การอ้างอิงเอกสาร

bizdocgen รองรับการเชื่อมโยงเอกสารแบบลูกโซ่:

```
ใบเสนอราคา (QT-001)
    ↓
ใบแจ้งหนี้ (INV-001) → อ้างอิง QT-001
    ↓
ใบเสร็จ (RC-001) → อ้างอิง INV-001
```

### การตั้งค่าใน Grist

ใช้ Reference column ใน Grist เพื่อสร้างความสัมพันธ์:

1. เปิดตาราง Documents
2. เพิ่มคอลัมน์ `Reference`
3. ตั้งค่าเป็น Reference ไปยังเอกสารอื่น

## การพิมพ์แต่ละประเภท

แต่ละประเภทเอกสารมีการจัด layout ที่เหมาะสม:

| ประเภท | Layout พิเศษ |
|--------|-------------|
| Quotation | เน้นรายละเอียดราคา |
| Invoice | แสดงข้อมูลการชำระเงิน |
| Receipt | เน้นส่วนลงชื่อ |

## CSS Classes สำหรับแต่ละประเภท

สามารถใช้ CSS ปรับแต่งตามประเภทเอกสาร:

```css
/* ใบเสนอราคา */
[data-document-type="quotation"] .document__header {
  border-bottom: 3px double #333;
}

/* ใบแจ้งหนี้ */
[data-document-type="invoice"] .document__payment-section {
  background: #f0f8ff;
}

/* ใบเสร็จ */
[data-document-type="receipt"] .signature-area {
  border: 2px solid #000;
}
```
