# ฟีเจอร์

## ประเภทเอกสาร

bizdocgen รองรับเอกสารธุรกิจ 3 ประเภท:

### ใบเสนอราคา (Quotation)

- แสดงข้อมูลผู้ออกเอกสารและลูกค้า
- ตารางรายการสินค้า/บริการพร้อมราคา
- สรุปยอดรวมและภาษี
- ช่องลงชื่อ 2 ฝั่ง — ผู้เสนอราคาและผู้อนุมัติ
- แสดงเครดิตเทอม (ถ้ามี)

### ใบแจ้งหนี้ (Invoice)

- ข้อมูลครบถ้วนเหมือนใบเสนอราคา
- แสดงข้อมูลการชำระเงิน (บัญชีธนาคาร, พร้อมเพย์)
- QR Code พร้อมเพย์พร้อมจำนวนเงิน (ถ้ากำหนดค่าไว้)
- ช่องลงชื่อฝั่งเดียว (ผู้ออกเอกสาร)
- อ้างอิงเลขที่เอกสารอื่น (ถ้ามี)

### ใบเสร็จรับเงิน (Receipt)

- ยืนยันการรับชำระเงิน
- แสดงข้อมูลการชำระเงินและ QR Code
- ช่องลงชื่อฝั่งเดียว (ผู้รับเงิน)
- จำนวนเงินรวมแสดงเป็นตัวอักษรภาษาไทย (เช่น "สองแสนแปดหมื่นบาทถ้วน")

## การคำนวณภาษี

ระบบรองรับการคำนวณภาษี 2 รูปแบบ โดยกำหนดค่าในฟิลด์ `Tax`:

### VAT 7%

กำหนดค่า `Tax` เป็น `0.07`:

- แสดงป้ายกำกับ "VAT 7%"
- คำนวณ: ยอดรวมสินค้า × 0.07
- ยอดสุทธิ = ยอดรวมสินค้า + VAT

### ภาษีหัก ณ ที่จ่าย (WHT)

กำหนดค่า `Tax` เป็นตัวเลขติดลบ เช่น `-0.03` สำหรับ 3%:

- แสดงป้ายกำกับ "ภาษีหัก ณ ที่จ่าย 3%"
- คำนวณ: ยอดรวมสินค้า × (-0.03)
- ยอดสุทธิ = ยอดรวมสินค้า − ภาษีหัก ณ ที่จ่าย

!!! example "ตัวอย่างอัตราภาษี"
    | ค่า `Tax` | ความหมาย |
    |-----------|----------|
    | `0.07` | VAT 7% |
    | `0` | ไม่มีภาษี |
    | `-0.03` | หัก ณ ที่จ่าย 3% |
    | `-0.05` | หัก ณ ที่จ่าย 5% |

## QR Code พร้อมเพย์

เมื่อกำหนดหมายเลขพร้อมเพย์ในข้อมูลวิธีการชำระเงิน (`Payment_Method.PromptPay`) ระบบจะสร้าง QR Code อัตโนมัติ โดย:

- รองรับหมายเลขโทรศัพท์ 10 หลัก หรือเลขประจำตัวผู้เสียภาษี
- ฝังจำนวนเงินที่ต้องชำระลงใน QR Code
- แสดงบนเอกสาร ใบแจ้งหนี้ และ ใบเสร็จรับเงิน

## รองรับ Markdown

ฟิลด์ต่อไปนี้รองรับรูปแบบ Markdown:

- **รายละเอียดสินค้า** (`Items.Description`) — รองรับตัวหนา รายการ และโค้ด
- **หมายเหตุ** (`Remarks`) — รองรับ Markdown เต็มรูปแบบ

!!! example "ตัวอย่าง Markdown ในรายละเอียดสินค้า"
    ```markdown
    **บริการพัฒนาเว็บไซต์**
    - ออกแบบ UI/UX
    - พัฒนา Frontend ด้วย Vue 3
    - `responsive design`
    ```

!!! warning "การป้องกันความปลอดภัย"
    ระบบมีการ Sanitize เนื้อหา Markdown โดยจะลบ `<script>`, `<iframe>` และ Event Handler เพื่อป้องกัน XSS

## การพิมพ์เอกสาร

เอกสารถูกออกแบบให้พิมพ์ได้สมบูรณ์แบบในขนาด A4:

- **ขนาดกระดาษ**: A4 (210mm × 297mm)
- **ระยะขอบ**: 20mm ทุกด้าน
- **ฟอนต์**: Sarabun (รองรับภาษาไทย)
- **หน่วยวัด**: ใช้ pt สำหรับตัวอักษร และ mm สำหรับเลย์เอาต์ เพื่อความแม่นยำในการพิมพ์

เมื่อสั่งพิมพ์ ระบบจะ:

1. ซ่อนปุ่มกดและแถบเครื่องมือ
2. ซ่อนแผงตั้งค่า
3. รักษาสีพื้นหลังและสีตัวอักษร
4. แสดงเฉพาะเนื้อหาเอกสาร

## ข้อมูลการชำระเงิน

รองรับการแสดงข้อมูลการชำระเงิน:

- **บัญชีธนาคาร** — ชื่อธนาคาร สาขา เลขที่บัญชี ชื่อบัญชี
- **พร้อมเพย์** — หมายเลขพร้อมเพย์พร้อม QR Code

## ฟีเจอร์เพิ่มเติม

### เลขอ้างอิง

สามารถอ้างอิงเลขที่เอกสารอื่นได้ เช่น ใบแจ้งหนี้อ้างอิงเลขที่ใบเสนอราคา

### เครดิตเทอม

กำหนดระยะเวลาการชำระเงิน เช่น "Net 30" ผ่านฟิลด์ `Credit_Term`

### คัดลอก JSON

กดปุ่ม **📋 คัดลอก JSON** เพื่อคัดลอกข้อมูลดิบจาก Grist ในรูปแบบ JSON เหมาะสำหรับการดีบัก

### เอกสารที่ลงชื่อแล้ว

เมื่อกำหนด URL ของเอกสารที่ลงชื่อแล้ว (`Signed_Document_URL`) ระบบจะ:

- แสดงข้อความ "เอกสารนี้ได้ถูกลงชื่อเรียบร้อยแล้ว"
- แสดงลิงก์ดาวน์โหลดแทนตัวอย่างเอกสาร
- ปิดใช้งานปุ่มพิมพ์

### จำนวนเงินเป็นตัวอักษร

ยอดรวมสุทธิจะแสดงเป็นตัวอักษรภาษาไทยอัตโนมัติ เช่น:

> สองแสนแปดหมื่นสองพันสี่ร้อยห้าสิบสาม บาทเก้าสิบสองสตางค์
